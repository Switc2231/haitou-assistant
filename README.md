# 海投助手 v0.4.7

## ⚠️ 使用说明与风险提示

本项目是一个本地浏览器求职辅助插件，主要用于学习 Chrome Extension、DOM 自动化，以及个人求职流程优化。

本项目不提供云服务。项目作者不通过自建服务器采集、上传或出售任何用户数据；插件配置、简历文字、图片等信息保存在用户本地浏览器环境中。用户主动启用 DeepSeek AI 或确认投递时，相关内容会按功能需要直接发送给 DeepSeek 或对应招聘平台，具体范围见 [PRIVACY.md](./PRIVACY.md)。

请仅将本项目用于学习研究和个人本地辅助使用。请勿将本项目用于批量骚扰、刷屏、绕过平台风控、规避验证码、采集平台数据、多账号自动化操作、商用代投服务或任何违反第三方平台规则的用途。

使用者应自行了解并遵守 BOSS 直聘等第三方平台的用户协议、社区规则和相关法律法规。因不当使用导致的账号限制、封禁、投诉、法律纠纷或其他后果，由使用者自行承担。

本项目不保证适配任何第三方平台的未来页面变化，也不保证使用过程中不会触发平台风控。遇到验证码、异常页面、405 页面、账号风险提示或其他异常情况时，请立即停止使用。

一个面向 BOSS 直聘的 Chrome / Edge 开源扩展：收集岗位、本地或 AI 辅助筛选、人工审核后逐个进入聊天，并发送简历图片和可选的自我介绍。

本项目基于 MIT 项目 [JobCopilot](https://github.com/huluobo2237-pixel/JobCopilot) 二次开发，与 BOSS 直聘官方无关。

## 功能

- 按多个岗位关键词和城市收集 BOSS 岗位。
- 岗位去重、本地关键词筛选或可选的 DeepSeek AI 筛选。
- 投递前必须由用户勾选岗位，单轮最多处理 5 个。
- 从岗位详情建立沟通并校验对应聊天会话。
- 发送简历图片，以及 AI 生成或用户自己填写的自我介绍。
- 防止同一会话重复发送简历图片或完全相同的文字。
- 图片已发送但文字失败时标记“待人工确认”，不盲目重试。
- 保存岗位池、投递状态，支持 CSV 导出。

## 发送规则

| 当前配置 | 实际行为 |
| --- | --- |
| 已启用 DeepSeek，且 API Key 与简历文字完整 | 根据简历和岗位 JD 生成自我介绍，安全校验通过后发送 |
| 未启用 AI，但填写了完整的“无AI自我介绍” | 原样发送用户填写的文字 |
| 无有效文字，但已上传简历图片 | 只发送简历图片，不附带插件默认话术 |
| 图片和有效文字都没有 | 在联系招聘方之前安全停止 |

仍含 `【占位内容】` 的自我介绍不会发送。插件/API设置提示会在后台和聊天页被双重拦截。

## 安装

第一次使用建议先阅读：[完整安装使用教程](./海投助手使用教程/使用教程.md)。压缩包内同时提供 Word 版教程，方便转发给普通用户。

1. 下载并解压发布 ZIP，不能直接在压缩包里加载。
2. Chrome 打开 `chrome://extensions/`；Edge 打开 `edge://extensions/`。
3. 开启“开发者模式”。
4. 点击“加载已解压的扩展”。
5. 选择里面直接能看到 `manifest.json` 的文件夹。
6. 点击扩展图标打开侧栏。

如果电脑里还启用了其他 BOSS 自动投递扩展，请先关闭，避免多个扩展同时操作同一页面。

## 使用

1. 上传自己的简历图片。
2. 填写岗位关键词；城市留空时按全国搜索。
3. 可按求职方向填写本地加分词和排除词，不需要修改源码。
4. 如需 AI 增强，填写自己的 DeepSeek API Key 和简历文字。
5. 未接 AI 时，可自行填写“无AI自我介绍”；留空则不发送插件文字。
6. 点击“开始找岗位”，审核并勾选结果。
7. 第一次只选择 1 个岗位测试，确认图片、文字和会话均正确后再继续。

## AI 与数据

- 当前公开版只直接支持 DeepSeek API，默认模型为 `deepseek-chat`。
- AI 增强默认关闭；不填写 API Key 也可以使用本地规则。
- 启用 AI 后，简历文字和岗位信息会直接发送给 DeepSeek，用于筛选和生成自我介绍。
- API Key、配置、岗位池和投递记录保存在浏览器扩展本地存储中；项目没有自建中转服务器和统计服务。
- 简历图片会通过 BOSS 页面上传给招聘方。

详细说明见 [PRIVACY.md](./PRIVACY.md)。

## 安全边界

- 这是辅助工具，不保证投递结果，也不替代人工审核。
- 不绕过验证码、登录验证、平台限流或账号风控。
- 遇到 405、验证码、异常弹窗或页面结构变化时立即停止。
- BOSS 页面更新后，DOM 选择器可能失效；出现异常时不要继续批量操作。
- 更新代码后必须在扩展管理页点击“重新加载”。

## 开发与测试

```powershell
npm test
```

```powershell

node .\tests\chat-guard-smoke.js
```

测试通过后仍应使用测试账号或单岗位做人工观察，不要直接批量验证。
